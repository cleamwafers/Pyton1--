エラーが出たときにすること

  1. まず、気持ちを落ち着ける。
  2. エラーをよく見る。
  3. 修正を行う。

ダイアログで表示されるエラー

  • 単純な入力ミスや打ち間違いやインデントがずれていないかを確認する。
    1. 『invalid syntax』『invalid character』

      2. 『無駄な命令文があります』、『無効な文字があります』というエラー

      以下に注意する

      変数や関数の『名前のスペル』を間違えていないか

      『大文字と小文字』を間違えていないか

      『半角英数文字と全角文字』を間違えていないか

      『形の似た文字』を間違えていないか

      形の似た文字の例

      『.(ピリオド)』と『,(カンマ)』

      『o(オー)』と『0(数字のゼロ)』

      『l(小文字のエル)』『I(大文字のアイ)』

      『z(ゼット)』と『2(数字のニ)』

      『B(ビー)』と『8(数字のハチ)』

    2. 『expected』

      3. 『if文やfor文やdel文の行末にあるはずの:が見つかりません』というエラー

    3. 『unexpected indent』

      4. 『不要なインデントがあります。』というエラー

    4. 『expected an indented block』
      『unindent does not match any outer indentation level』

      5. 『インデントにズレがあります。』というエラー

    5. 『unterminated string literal』

      6. 『文字列の最後(文字列を閉じる側のダブルクォーテーション[""]またはシングルクォーテーション['']が足りない)が綴じられていません』というエラー

    6. 『'('was never closed 』『unmatched')'』

      7. 『綴じカッコ[丸かっこ()や角かっこ[]]がありません(または多すぎます)』というエラー

    赤い文字で表示されるエラー

  • 多くの場合がうっかり書き間違えたようなミス。
  • 肝心なのはエラー表示の一番最後の行。
    1. 『NameError; name 'xxx' is not defined』

      2. 『この名前の変数名や関数名が見つかりません』

      以下を確認しましょう。

      エラーの場所の『変数名』や『関数名』を間違えていないか
      例:printがPrintになってしまっている。

      『変数や関数を作ったとき』に名前を間違えていないか→遡って変数や関数を作ったときの名前を確認する。
      例:変数名『bmi』を間違えて『bm1』と書いてしまった場合。

      タートルプログラムのファイル名を『turtle.py』にしていないか
      そのフォルダ内にライブラリと同じ名前のファイルがあると正しく動かなくなるので同じ名前にしない。
      例:NameError: name 'shape' is not defined 'shape'という名前の命令を知りません。という意味

    2. 『ModuleNotFondError: No module named 'xxx'』

      3. 『その名前のモジュール(ライブラリ)が見つかりません』というエラー

      以下に注意する

      『ライブラリ名』を間違えていないか→大文字や小文字が間違って居るだけでもエラーになる。
      例①:ModuleNotFoundError: No module named 'pil'→『'pil'という名前のライブラリが見つかりません。』という意味。
      原因『PIL.Image』を間違えて『pil.Image』と書いている
      例②:ModuleNotFoundError: No module named 'PIL.ImageTK'→『'PIL.ImageTK'という名前のライブラリが見つかりません。』という意味。
      原因『PIL.ImageTk』を間違えて『PIL.ImageTK』と書いている。

      ライブラリを正しくインストールしているか→ライブラリ名は正しいのにエラーが出るときはライブラリが正しくインストールされていない可能性がある。
      Python言語は『一つのパソコンの中に複数のバージョンをインストールすることができる』

    3. 『FileNotFoundError: [Errno 2]No such file or directory: 'xxx'』

      4. 『読み込もうとしたら、ファイルが見つかりませんでした』というエラー

      そのプログラムのフォルダ内に指定したファイルがあるか確認する。なければファイルを用意する。
      例①:FileNotFondError: [Errno 2]No such file or directory: '2.png'→『'2,png'というファイルかフォルダが見つかりません。』という意味。
      原因プログラムで読み込む『2,png』ファイルがない場合

    4. 『TclErrpr: bad geometry specifier "xxx"』

      5. 『tkinterのパラメータが間違っています』というエラー

      tkinterではtkinterが決めた記号でしか正しく動きません。例:『root.geometry("200x100")』の『x』の記号には『半角英数字の小文字のエックス』を使ってください。

      _thinter.TclError: bad geometry specifier "200X100"
      意味:『tkinterの'200X100'というパラメータが間違っているようです。』

    5. 『ValueError:'Grays'is not a valid value for name』

      6. 『指定された名前のパラメーターが間違っています』というエラー

      ValueError:'Grays'is not a valid value for name;
      意味:『'Grays'という名前のパラメータが間違って居るようです。』 原因:『Greys』を間違えて『Grays』と書いている場合
